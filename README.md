# Клиент-Серверное приложение

Тестовое задание: минимальное клиент-серверное приложение на C# с использованием SQLite, Web API и Blazor.

## Техническое задание

### Серверное приложение (ASP.NET Core Web API)
- **Порт**: 8080
- **База данных**: SQLite с Entity Framework Core
- **Фоновый сервис**: Добавляет записи каждую минуту
- **API эндпоинт**: `GET /records` - возвращает все записи в формате JSON

### Клиентское приложение (Blazor Server)
- **Главная страница**: Отображает время сервера 
- **Страница базы**: Отображает данные из БД в таблице
- **Стили**: Bootstrap для оформления интерфейса

##  Технологии

### Сервер
- **C#**, .NET 8
- **ASP.NET Core Web API**
- **Entity Framework Core** + SQLite
- **Newtonsoft.Json**

### Клиент  
- **C#**, .NET 8
- **Blazor Server**
- **Razor Pages**
- **Bootstrap**
- **Kestrel** веб-сервер

##  Запуск приложения

### Предварительные требования
- [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
- Visual Studio 2022 или VS Code

### Инструкция по запуску

1. **Клонируйте репозиторий**
   ```bash
   git clone <url-репозитория>
   cd <папка-проекта>
   ##  Запуск через Visual Studio

1. **Откройте решение в Visual Studio 2022**
   - Запустите Visual Studio
   - Выберите "Open a project or solution"

2. **Запустите серверное приложение**
   - В окне Solution Explorer правой кнопкой мыши на проекте **Server**
   - Выберите "Set as Startup Project"
   - Нажмите Ctrl+F5 (запуск без отладки) или F5 (с отладкой)
   - Сервер запустится на `https://localhost:8080`

3. **Запустите клиентское приложение**
   - Правой кнопкой на решении → "Properties"
   - В "Startup Project" выберите "Multiple startup projects"
   - Установите для **Server** и **Client** действие "Start"
   - Нажмите Apply и OK
   - Запустите решение Ctrl+F5
   - Клиент откроется на `https://localhost:7182`

##  Проверка работоспособности

После запуска:

1. **Откройте главную страницу** (https://localhost:7182)
   - Должно отображаться время сервера
   - Статус "Подключение установлено"

2. **Перейдите на страницу базы данных** (https://localhost:7182/database)
   - Должна отображаться таблица с записями
   - Новые записи добавляются автоматически каждую минуту

3. **Проверьте API** (https://localhost:8080/swagger)
   - Должен работать эндпоинт GET /records
   - Должен возвращать JSON с данными
